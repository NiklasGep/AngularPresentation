Komponenten

Was versteht man unter einer Komponente

Eine Angular-Komponente ist eine eigenständige Einheit innerhalb einer Angular-Anwendung, die eine bestimmte Funktionalität bereitstellt und eine klare Trennung von Verantwortlichkeiten ermöglicht. Eine Komponente besteht aus einer Klasse, die die Logik der Komponente enthält, sowie aus einer Vorlage, die das Aussehen der Komponente definiert. Die Vorlage enthält HTML-Code, der durch Angular-Direktiven erweitert werden kann, um dynamische Daten anzuzeigen oder Benutzereingaben zu verarbeiten.

Komponenten können über Eingabe- und Ausgabeparameter (Bindings) miteinander interagieren, um Daten zwischen verschiedenen Komponenten auszutauschen. Dadurch können Entwickler komplexe Anwendungen aus mehreren Komponenten aufbauen und diese miteinander kombinieren(Services).

Komponenten sind eine grundlegende Einheit in der komponentenbasierten Architektur von Angular und werden häufig verwendet, um spezifische Funktionalitäten wie z.B. Navigation, Authentifizierung oder Datenanzeige zu implementieren. Sie können auch wiederverwendet werden, um ähnliche Funktionalitäten in verschiedenen Teilen der Anwendung bereitzustellen.

Erstellen einer Komponente

Eine Komponente erstellt man durch das eintippen folgendes Befehls in der Konsole:

ng generate component my-component

Durch diesen Befehl wird dann eine neue Komponente in dem 'src/app'-Ordner erstellt und hinzugefügt. Außerdem wird die Komponente automatisch in der App-Module importiert und kann somit im ganzen Projekt durch einen simplen Import verwendet werden

Verwendung von Komponenten

Komponenten lassen sich ganz leicht durch HTML-taggs verwenden. In der TypeScript einer Komponente finde ich unter "selector" den tag, welchen ich in der HTML zum aufrufen benutzten kann.

Services

In Angular ist ein Service eine Klasse, die eine spezifische Funktionalität bereitstellt und von Komponenten und anderen Services verwendet werden kann. Ein Service kann Daten von einer API abrufen, komplexe Geschäftslogik durchführen oder Daten lokal im Speicher speichern.

Services sind in Angular ein wichtiger Bestandteil der komponentenbasierten Architektur, da sie die Aufgaben von der Darstellung von Daten trennen und somit die Wiederverwendbarkeit und Wartbarkeit von Code verbessern. Eine Service-Klasse wird normalerweise als Singleton-Instanz bereitgestellt, was bedeutet, dass es immer nur eine einzige Instanz des Dienstes gibt, die von allen Komponenten verwendet wird.

Insgesamt bieten Services in Angular eine effektive Möglichkeit, Funktionalitäten zu organisieren und zu teilen, die von mehreren Komponenten oder anderen Teilen der Anwendung genutzt werden.
